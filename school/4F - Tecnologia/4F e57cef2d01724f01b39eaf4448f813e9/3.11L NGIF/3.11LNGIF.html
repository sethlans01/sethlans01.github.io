NGIF

In Angular esistono delle classi chiamate direttive, che permettono di modificare il comportamento dei componenti e dei loro elementi.

Si suddividono in 3 categorie:

  • Built-in: modificano il comportamento degli elementi;
  • Attribute: modificano il comportamento o l’apparenza degli elementi;
  • Structural: cambiano il DOM, aggiungendo o rimuovendo elementi.

La prima direttiva che vediamo è NGIF, che permette di mostrare o no un componente in base al risultato di un IF.

Costrutto base

Vogliamo aggiungere un elemento che viene mostrato solo se una determinata condizione si verifica.

L’elemento che vogliamo aggiungere è:      <p> Condizione verificata </p>

Per poter fare ciò occorre aggiungere al tag HTML dell’elemento il costrutto NGIF, come mostrato nella seguente figura.

Nella parte destra dell’uguale si inserisce una condizione booleana, all’interno delle virgolette.

Se la condizione booleana è vera, allora l’elemento viene mostrato nella pagina.

Se la condizione booleana è falsa, allora l’elemento NON viene mostrato nella pagina.

Inoltre, occorre anche importare nel file Typescript il modulo chiamato CommonModule.

import { Component } from '@angular/core';

import { CommonModule } from '@angular/common';

@Component({

 selector: 'first-component',

 standalone: true,

 imports: [CommonModule],

 templateUrl: './first-component.component.html',

 styleUrl: './first-component.component.css'

})

export class FirstComponentComponent {

}

È possibile usare anche delle variabili definite nel file Typescript del componente.

FILE HTML:

<p *ngIf = "isVisible"> Condizione verificata </p>

FILE TYPESCRIPT:

import { Component } from '@angular/core';

import { CommonModule } from '@angular/common';

@Component({

 selector: 'first-component',

 standalone: true,

 imports: [CommonModule],

 templateUrl: './first-component.component.html',

 styleUrl: './first-component.component.css'

})

export class FirstComponentComponent {

 isVisible = true

}

Aggiungere il ramo else

Come tutti i costrutti if, anche NGIF permette di creare un ramo else.

Per poter vedere come si fa, dobbiamo prima introdurre un nuovo tag HTML.

Angular mette a disposizione un tag HTML speciale, chiamato <ng-template>.

Tutto quello che viene messo al suo interno viene ignorato dal browser e da HTML, ma viene utilizzato solo da Angular.

Per vedere come funziona il ramo else del costrutto NGIF, partiamo subito con un esempio.

FILE HTML:

<p *ngIf = "isVisible; else bloccoElse"> Condizione verificata </p>

<ng-template #bloccoElse>

   <p> Condizione non verificata </p>

</ng-template>

Per aggiungere il ramo else si mette un punto e virgola dopo la condizione booleana di NGIF e si scrive else bloccoElse, dove bloccoElse può essere qualsiasi stringa ed è il nome con il quale noi andiamo a indicare il blocco di codice da mostrare nel caso in cui si entri nel ramo else di NGIF.

Per dichiarare il blocco di codice da visualizzare quando il ramo else viene preso, lo si inserisce all’interno del tag  <ng-template>.

Aggiungere il blocco then

Il fatto che Angular ci permetta di definire ciò che viene visualizzato nel caso in cui si prenda il ramo else del costrutto NGIF usando più elementi, inseriti all’interno del tag <ng-template> è molto utile.

Si può fare la stessa cosa anche per definire il codice che viene visualizzato se il costrutto NGIF risulta vero, usando il blocco then.

<ng-template *ngIf = "isVisible; then bloccoIf else bloccoElse"></ng-template>

<ng-template #bloccoIf>

   <p> Condizione verificata </p>

</ng-template>

<ng-template #bloccoElse>

   <p> Condizione non verificata </p>

</ng-template>

Il codice da visualizzare si inserisce all’interno di un altro tag <ng-template> e poi si inserisce all’interno della stringa che dichiara il costrutto NGIF then bloccoIf, dove bloccoIf è una stringa qualsiasi e indica il nome del blocco di codice che viene visualizzato quando l’NGIF è vero.

Quando si vuole specificare il blocco then all’interno dell’NGIF, l’NGIF si può dichiarare dentro un tag <ng-template>.

NB: se non lo si fa (ma ad esempio vogliamo continuare a usare il tag <p> dell’esempio di prima), il tag che definisce l’NGIF non verrà MAI visualizzato sulla pagina web!
