Git hub

Creare un repository su git hub

Per prima cosa dovete aprire un account su https://github.com con l’email scolastica e cognome_nome come username

 e creare un nuovo repository

Impostate il repository come in figura

Dovreste riuscire ad ottenere l’indirizzo del vostro repository

Clonare il repository in locale

Create una cartella vuota e accedetevi con il prompt dei comandi o con la git bash (vedi esercizio 0).

Il comando “git clone[percorso_su_git_hub]” trasferisce tutto il contenuto del vostro repository su git hub sul vostro pc (in questo caso ho creato una cartella progetto_git_test sulla mia chiavetta [F:\])

A questo punto dovreste trovare dentro alla vostra cartella lo stesso file readme che git hub aveva automaticamente inserito nel vostro progetto

Identificarsi al repository locale[1]

A questo punto avete due repository: locale e remoto.

Il file Readme.md viene dunque triplicato.

Nella Working copy, sul repository locale e sul repository remoto.

Creare un progetto e aggiungerlo al repository

Tutti i file che creerete nella cartella del repository (primo_repo_test) potranno essere aggiunti al vostro repository remoto.

0) Crea un file all’interno della cartella

1) Dite a git di aggiungere tutti i file (*) all’indice delle modifiche (staging area)

2) Ora confermate (commit -m “messaggio”) le modifiche al repository locale

3)Riportate le modifiche sul repository remoto

git push origin main

A questo punto dovreste trovare su git hub tutti i file che avete in locale

Aggiornare le modifiche

Supponete di modificare un file

Se volete riportare le modifiche anche su github, dovete procedere come segue

  • Aggiungete i file alla staging area (* aggiunge tutti i file)

 

  • Committate sul repository locale

  • Fate una push sul repository remoto

Simuliamo che qualcuno modifichi il nostro progetto

Andate su git hub ed editate il vostro file HTML (simulando che qualcun altro abbia modificato lo stesso programma che avete creato voi)

A questo punto potete riportare le modifiche sul vostro repository con il comando git pull origin master

  • Origin è il nome che git da’ in automatico al repository remoto (possono essercene più di uno, per listarli tutti usate il comando git remote -v)

  • Master è il branch principale del vostro progetto. Non abbiamo ancora parlato di branch e quindi ne avete uno solo per ora. (per listare tutti i branch usate il comando git branch -v)

 (in realtà pull è l’unione di due comandi che per ora non vediamo fetch e merge)

ESERCIZIO

Git può gestire più repository remoti alla volta, il comando git remote add permette di aggiungere un repository alla cartella git locale.

1 - Crea un repository su git hub

2 - Aggiungilo al repository locale su cui stai lavorando (git remote add origin2 <url nuovo repository>

3 - Committa tutte le modifiche locali

4 - Invia le modifiche al nuovo repository (git push origin2 master)

Sotto trovate un riassunto dei comandi di git (in rosso i comandi che userete più spesso)
